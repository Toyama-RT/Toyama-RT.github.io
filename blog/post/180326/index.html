そこで、180326となっているタイトルをどうしてこうなったのか　に変更した。 またdraftをtrueからfalseに変更した。こうするとローカルホストで表示されるようになる。

この間、失敗したがエラーメッセージなどは全く表示されず、コマンドプロンプトを見て判断するしかない。表示二は気をつけた方がいい。また参考になるサイトを見ながらやってもイマイチ曖昧なところが多い。 ひとつひとつ積み上げていくしかないようだ。

現在困っているのは、ロゴマークがうまく表示されないところだ。serverでは表示できないし、サイトの構築をしても表示されない。もしかしてこれらが自動的にやってくれるというのは、思い込みでしかないのかもしれない。 なんとんかうそのような結末がちらちらと見えてきた。だんだん未来が読めるようになって来たのかもしれない。

ロゴが表示できるようになった。 方法　staticの中にimagesを作りなかに画像ファイルを入れる。

configファイルに logo = “/images/logo.jpg” と記載すれば、そこを見に行く。実に簡単にできているのに説明が見つからない。 今回は http://d.hatena.ne.jp/shouh/20160108/1452261442 shouhの日記 からこの人のサイトは実にわかりやすく書いてある。苦労のほどが偲ばれる。

リスト Hugo の Markdown ではspace4つである。なんてことも書いてある。

  • リスト
    • サブリスト
      • サブサブリスト

色々あるらしい。今回使用したのはrogo-zenだがいろいろ苦労が多い。 FrontMatterにカテゴリを設定する機能もある様だ。

ここまでくると形が見えてくる。 フッターが自動的にいい位置に来るなんてあたりまえにできている。 表示画面が小さくて縦長になってもきちんと表示される。 基本的な機能は既に全て（と思うが）そろっているこの完成度は見過ごせない。

しばらくこのような手順でいじり倒しておいこうかと思う。

それにしてもhtml生成がおかしい。 完成度がイマイチ。ロゴの参照元がうまくない。

困ったな。もっと有名どころが必要なのか。それともこのパソコンの問題なのだろうか？
